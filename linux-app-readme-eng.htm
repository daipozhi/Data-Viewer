Data Viewer 
     Data Viewer is open source project , is a database and user interface design tools , supported OS is windows XP/Vista/7/10 or
LINUX ,you can input,search,print your data , Data Viewer create UI by a text model file , including control item , 
it save data in standerd DBF file ,it have a lot examples , support link between database file , it can print data in order of a text model 
file , it support multiple users's operation in same time .



Data Viewer User Guide


1,design database structure file 

write a text file like this:

date f1 c 8
dept f3 c 40
topc f5 c 40
cont f4 m 8

first field is 'field description'
second field is 'field name'
third field is 'field type' (c char , n number )
4th field is 'field length'
     

save this file to hard drive , file name is *.tb3 , when program open this file ,
it will create a database file ,it is a standerd DBF database file .



2, user interface design

use char '-'   '|'   '+'   to write a table , use field alternative symbols in field's position ,
&? is memory field , &?f1 is memory field named f1 ,
&@ is button , &@a is a button named a ,
&( is radio button , &(1 is a radio button named 1 ,
&[ is checkbox button , &[1 is a checkbox button name 1 ,
&: is for database define , if 't2' is a database file's name , &:t2 means t2 database file ,
                            if 't2' is a field name , &:t2 means field of t2 .

3, define button's action

if your windows file have button , button's action can define like this :


@a button="[]" com=exit         (button a is exit)
@c button="[]" com=page_up      (button b is page up)
@b button="[]" com=page_down    (button c is page down)
@d button="[]" com=insert_rec   (button d is insert record )
@e button="[]" com=cancel       (button e is cancel )
@f button="[]" com=delete_rec   (button f is delete record )
@g button="[]" com=rollback     (button g is database rollback )
@h button="[]" com=commit       (button h is database commit )
@i button="[]" com=findcondition(button i is input search option )
@j button="[]" com=beginfind    (button j is start search )
@k button="[]" com=endfind      (button k is stop search )

6, define memory field 

you can write a window file like this :

    +---------------------------------+
    | Find Condition:                 |
    +----------+----------------------+
    |Date      |&?f1                  |
    +----------+----------------------+
    |Department|&?f3                  |
    +----------+----------------------+
    |Topic     |&?f5                  |
    +----------+----------------------+
    |  &@a OK            &@b Cancel   |
    +---------------------------------+

&?f1 means memory field named f1 , &?f3 means memory field named f3 ,

and you can write a text file like this :

?f1 e c 40 0
?f2 e c 3  0
?f3 e c 40 0
?f4 e c 4  0
?f5 e c 70 0
?f6 e c 70 0
?f7 e c 70 0
?f8 e c 20 0

save this text file to hard disk named *.tb2 ( same name with it's window file , extension name is *.tb2 )

f1,f2,f3... is memory field's name ,'e' means you can edit this field , 'c' is this field's data type ,
 40 is this field's lenght , 0 is decimals (for number data type field )

this window is for input search option , when you click 'FindOpt' button , it will create this window ,
 you can input some search option , and click 'StartFind' button , it will search data in database ,
 and display results .

after menu ,is 'window interface',at this time ,there are two windows ,input 's'(return) to next window,


in this window interface ,< a  Info A    > , < b  Info B    > mean 'Button' ,

< a Info A   > is button name a, 
< b Info B   > is button name b,
< g Info G   > is button name g,

input '< a'(return) mean click button a,
input '< b'(return) mean click button b,

 '+>+>+>'  mean 'Cursor',

 'u'(return) mean cursor move up,

 'd'(return) mean cursor move down,

 'r'(return) mean cursor move right,

 'l'(return) mean cursor move left,

 'c'(return) mean click mouse.

 's'(return) mean switch windows.


input '< c'(return) ,mean click button < c Info C   >,will display,
